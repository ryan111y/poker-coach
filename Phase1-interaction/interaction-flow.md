### 边角路径

```
1. 用户输入不完整
   手牌没选完 → "开始模拟"按钮置灰
   行动没填完 → 提示"请完成行动记录"

2. 模拟出错
   显示错误提示 → "服务异常，请重试"
   回到决策点选择页

3. 用户中途放弃
   点击 ← 返回 → 回到上一页
   刷新页面 → 回到首页

4. 快速示例
   点击示例卡片 → 自动填充预设牌局
   直接跳到决策点选择 → 省略手动输入
```

## 四、交互规则总结

### 导航规则
- 所有页面右上/左上都有 ← 返回按钮
- 返回时保持输入数据
- 步骤指示器（1/4、2/4）让用户知道在哪个阶段

### 数据规则
- 选手牌：必须选2张，不能重复
- 选公牌：可选0-5张，按位置排序
- 填行动：至少填翻牌前+翻牌的行动
- 决策点：至少要选一个替代方案才能模拟

### 反馈规则
- 操作立刻响应（点击高亮、按钮状态变化）
- 模拟期间有进度条
- 错误有明确提示
- 成功有明确结果

### 移动端规则
- 所有页面在手机宽度（375px）上可操作
- 按钮 ≥ 44px
- 选牌器全屏弹出
- 结果对比在手机上改为上下布局
