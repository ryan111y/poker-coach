# Phase 0: 产品定义

> 德扑复盘教练 · AI 陪练 + 平行宇宙复盘

---

## 一、痛点分析

### 核心痛点
德扑玩家最大的三个问题：
1. **复盘** — "我当时该不该跟？" "如果换一种打法会怎样？"
2. **练习** — 约不到人打，真人局输了要等下一手
3. **针对性训练** — 想练应对特定风格（紧凶/松凶/疯狂）

### 用户场景
```
场景A：晚上睡不着，AK翻牌中顶对你下注被跟注。
"如果我翻牌 check 而不是下注，EV 差多少？"
"他诈唬我的概率有多大？"

场景B：想练"怎么对抗 Tom Dwan 那种疯子"
线下的局没有 Tom Dwan，怎么办？
→ 开一局：AI Tom Dwan 坐对面，真人打法

场景C：明天要去跟牌友老张打，老张特别紧
想先热身：开一局 AI 版"老张"，先打打看
```

### 我们的回答
**两件事打一套：**
1. **复盘**：输入打过的牌 → 选决策点 → 模拟"如果换了会怎样"
2. **陪练**：选一个 AI 牌手 → 坐下来打一局 → 打完了自动复盘

## 二、AI 牌手系统

### 牌手怎么"活"起来
每个牌手不是标签，是一个**决策引擎**：

```
Tom Dwan
├── VPIP 32%        ← 玩多少牌
├── PFR 22%         ← 翻牌前加注频率
├── 3-bet 12%       ← 3-bet 频率
├── A.F. 4.5        ← 激进度（高=爱加注爱诈唬）
├── c-bet 72%       ← 翻牌持续下注率
├── Fold to c-bet 35%  ← 面对 c-bet 弃牌率
├── WTSD 28%        ← 看到摊牌率（低=诈唬多）
├── Bluff freq 高   ← 诈唬倾向
└── Position aware true ← 位置意识
```

每轮行动时，AI 不是随机打，而是：
1. 看自己手牌强度
2. 看底池赔率
3. 按牌手风格参数决定行动（跟注/加注/弃牌/诈唬）

### 牌手仓库

| 类型 | 来源 | 例子 |
|------|------|------|
| 🏆 职业选手 | 预置参数库 | Tom Dwan、谭轩、Phil Ivey |
| 👥 自定义牌友 | 从牌局生成 / 描述生成 | 老张、老李 |
| 🧬 进化牌手 | 记录多手后自动更新参数 | 老张(24手) |

### 牌手进化
每次复盘记录了牌手的一手行动 → 参数自动微调
> 10手 ≈ 基本摸清 → 50手 ≈ 很准 → 100手 ≈ 数字克隆

## 三、两大功能模块

### 模块 A：复盘分析（原 MVP）
```
输入牌局 → 选决策点 → Monte Carlo 模拟 → EV 对比
                              ↑
                         按牌手风格模拟
```

### 模块 B：AI 陪练（新增 ⭐）
```
选牌手 → 坐下打一局（轮流行动）→ AI 按风格决策 → 
打完自动进入复盘 → 看哪里没打好
```

## 六、功能优先级（更新）

### P0：MVP
```
□ 手牌输入 + 行动记录
□ AI 牌手模拟（职业选手 + 自定义）
□ Monte Carlo 模拟（按风格参数）
□ 复盘结果（EV 对比 + 胜率分析）
□ 牌手创建（从描述/牌局生成）
```

### P1：陪练模式
```
□ 选牌手 → 坐下打牌
□ AI 按风格决策
□ 打完自动进入复盘
□ 牌手进化（记录牌局更新参数）
```

### P2：
```
□ LLM 文字复盘总结
□ 牌友分享
□ 多个牌手同时对比
```
